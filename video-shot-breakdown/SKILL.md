# 短视频分镜拆解器

对用户上传的短视频进行深度分析，逐镜头拆解出最详细的分镜脚本，支持一比一复刻。

## 核心能力

- 支持**本地视频文件**和**URL链接**两种输入方式
- **优先直接视频分析**，完整捕捉运镜、转场、动作连续性、音频
- **备选关键帧分析**，适用于长视频或大文件场景
- 输出**JSON格式**分镜脚本，便于程序处理和二次开发
- 包含视频**主题、概要、情绪、类型、风格**等多维度分析
- 每个镜头提供**景别、运镜、画面、台词、音效、转场**等完整信息

---

## 分析流程

### 方案选择原则

| 视频情况 | 推荐方案 | 原因 |
|----------|----------|------|
| 时长 ≤ 2分钟 | **直接视频分析** | 完整捕捉运镜、转场、动作连续性、音频 |
| 时长 > 2分钟 | **关键帧分析** | 避免超出模型处理限制 |
| 文件较大（>100MB） | **关键帧分析** | 减少数据传输，提高处理速度 |

**优先使用直接视频分析**，因为能完整捕捉：
- 运镜方式（推/拉/摇/移/跟）
- 转场效果（硬切/淡入淡出/溶解等）
- 动作连续性和节奏
- 音频与画面的同步关系

---

### 方案一：直接视频分析（优先）

直接将视频传给多模态模型进行整体分析。

**优势：**
- 运镜判断准确（能观察镜头的推拉摇移过程）
- 转场检测精确（能直接看到画面切换效果）
- 动作连续完整（不断帧）
- 音画同步分析（台词、BGM、音效与画面对应）

**执行方式：**
1. 直接读取视频文件或URL
2. 将完整视频传给多模态模型
3. 模型一次性分析所有维度

**分析提示词模板：**
```
请分析这段短视频，逐镜头拆解出详细的分镜脚本。

需要识别的信息：
1. 视频整体：主题、概要、情绪、类型、风格
2. 每个镜头：时间码、景别、运镜方式、转场类型
3. 画面内容：场景、构图、光线、色彩、人物动作表情、服装道具
4. 音频内容：台词、旁白、背景音乐风格、音效
5. 剪辑节奏：镜头数量、平均时长、节奏快慢

请按照JSON格式输出完整的分镜脚本。
```

### 方案二：关键帧分析（备选）

当视频过长或文件过大时，提取关键帧进行分析。

**适用场景：**
- 视频时长 > 2分钟
- 文件大小 > 100MB
- 模型不支持视频输入时的降级方案

**关键帧提取模式**

提取脚本支持三种模式，默认使用 `hybrid`（推荐）：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `basic` | 按时间间隔均匀提取 | 视频节奏均匀、转场简单 |
| `scene` | 检测场景变化提取 | 转场明显的硬切视频 |
| `hybrid`（推荐） | 结合均匀间隔 + 场景变化，合并去重 | 所有场景，确保每个镜头都有代表帧 |

**关键帧提取策略**

| 视频时长 | 提取间隔 | 最大帧数 | 说明 |
|---------|---------|---------|------|
| < 15秒 | 每1秒1帧 | 15帧 | 短视频节奏快，需要密集采样 |
| 15-30秒 | 每1.5秒1帧 | 20帧 | 常见短视频时长 |
| 30-60秒 | 每2秒1帧 | 25帧 | 中等长度视频 |
| 60-120秒 | 每3秒1帧 | 30帧 | 较长视频 |
| > 120秒 | 每5秒1帧 | 40帧 | 长视频，需控制帧数 |

**混合模式工作原理**

```
通道 A（均匀间隔）     通道 B（场景变化）
  ▼                      ▼
  帧1 @0s                帧2 @3.1s（硬切）
  帧3 @4.5s              帧4 @7.8s（硬切）
  帧5 @9s
         ↓ 合并去重（时间差 < 间隔×0.6 视为重复）↓
  帧1 @0s → 帧2 @3.1s → 帧3 @4.5s → 帧4 @7.8s → 帧5 @9s
```

- 两个通道并行提取
- 时间差 < 间隔×60% 的帧视为重复，优先保留场景变化帧（转场点更精准）
- 超 max_frames 时优先保留场景变化帧，均匀间隔帧按比例采样

**自适应阈值**（`--adaptive`）

默认场景变化阈值 0.3 适用于大部分硬切视频。开启 `--adaptive` 后，脚本会先分析视频的场景复杂度，自动调整阈值：
- 快节奏/多转场视频 → 降低阈值（0.15-0.2），捕捉更多变化
- 慢节奏/少转场视频 → 保持阈值（0.25-0.3），避免假阳性
- 手持晃动视频 → 适当提高阈值，过滤抖动伪变化

**智能缩放**

根据原始分辨率自动选择输出尺寸，保留高清视频的服饰细节：

| 原始分辨率 | 缩放目标 | 说明 |
|------------|----------|------|
| 4K+ (3840+) | 2560px | 保留服饰纹理细节 |
| 2K (2560+) | 1920px | 保留大部分细节 |
| 1080p/720p (1280+) | 1280px | 标准处理 |
| < 720p | 不缩放 | 保留原始质量 |

**使用方法**
```bash
# 推荐方式：混合模式（默认）
python scripts/extract_frames.py video.mp4

# 混合模式 + 自适应阈值
python scripts/extract_frames.py video.mp4 --mode hybrid --adaptive

# 基础模式
python scripts/extract_frames.py video.mp4 --mode basic

# 场景变化模式 + 自定义阈值
python scripts/extract_frames.py video.mp4 --mode scene --threshold 0.2

# JSON 输出（含每帧时间码）
python scripts/extract_frames.py video.mp4 --output-json
```

**执行方式：**
1. 提取关键帧图片序列（混合模式默认）
2. 每帧携带精确时间码（用于镜头起止时间定位）
3. 将关键帧按时间顺序传给多模态模型
4. 模型基于关键帧推断镜头信息和画面内容
5. 注意：运镜和转场需要根据帧间差异推断，可能不够准确

### 真人出镜判断 (on_screen_presence)

| 字段 | 说明 |
|------|------|
| has_real_person | 是否有真人出镜（true/false） |

| 级别 | 占比 | 说明 |
|------|------|------|

| **低** | <30% | 人物仅偶尔出现，或露出面积很小（如手部特写、背影等） |

**person_details 字段说明：**

| 字段 | 说明 |
|------|------|
| person_id | 人物编号 |
| description | 人物描述（性别、年龄段、外貌特征） |
| screen_time_ratio | 出镜时间占比（0-1之间） |
| appearance_notes | 出镜情况说明 |

## 执行步骤

### 步骤1：判断分析方案

1. 获取视频基本信息（时长、文件大小）
2. 根据时长和大小选择分析方案：
   - 时长 ≤ 2分钟 且 文件 ≤ 100MB → **直接视频分析**
   - 其他情况 → **关键帧分析**

### 步骤2：执行分析

**直接视频分析流程：**
1. 直接将视频传给多模态模型
2. 模型同时分析画面和音频
3. 一次性获取完整的分镜信息

**关键帧分析流程：**
1. 使用 `hybrid` 模式提取关键帧（均匀间隔 + 场景变化双通道）
2. 脚本自动合并去重，每帧携带精确时间码
3. 将关键帧序列按时间顺序传给多模态模型
4. 根据帧间差异推断运镜和转场

### 步骤3：整合分析结果

1. 提炼视频主题和概要
2. 分析情绪基调和变化
3. 判断视频类型和风格
4. 整理镜头分割和转场信息

### 步骤4：生成JSON输出

按照JSON结构填充分析结果。

## 辅助脚本

### 关键帧提取脚本

本技能提供关键帧提取脚本 `scripts/extract_frames.py`。

**使用方法：**
```bash
python scripts/extract_frames.py <视频路径> [输出目录] [--mode <basic|scene|hybrid>] [选项]
```

**参数说明：**
- `--mode hybrid`（默认）: 混合模式，结合均匀间隔 + 场景变化，确保每个镜头都有代表帧
- `--mode basic`: 基础模式，按时间间隔提取
- `--mode scene`: 场景变化模式，检测场景变化提取关键帧
- `--adaptive`: 自适应阈值（hybrid/scene 模式），自动分析视频复杂度调整阈值
- `--threshold 0.3`: 手动设置场景变化阈值（默认 0.3）
- `--interval 2`: 手动设置提取间隔秒数（basic 模式）
- `--max-frames 30`: 最大提取帧数
- `--output-json`: 以 JSON 格式输出结果（含每帧时间码）

**依赖要求：**
- Python 3.6+
- ffmpeg（需预先安装）

## 注意事项

1. **方案选择**：优先使用直接视频分析，仅在视频过长或文件过大时使用关键帧分析
2. **关键帧模式**：推荐使用 `hybrid` 混合模式，确保每个镜头都有代表帧；开启 `--adaptive` 可自适应调整场景变化阈值
3. **分析精度差异**：
   - 直接视频分析：运镜、转场、动作判断更准确
   - 关键帧分析（hybrid）：均匀间隔 + 场景变化互补，减少漏检和误检
   - 关键帧分析（scene only）：渐变转场（溶解/淡入淡出）可能漏检
   - 关键帧分析（basic only）：镜头边界不精确，但时间覆盖完整
4. **视频质量**：视频清晰度越高，分析结果越准确；智能缩放会保留高清视频细节
5. **时长限制**：建议分析2分钟以内的短视频，直接视频分析效果最佳
6. **隐私保护**：用户上传的视频仅用于分析，不会被存储或用于其他用途
7. **分析准确性**：AI分析可能存在误差，关键信息建议人工复核

## 错误处理

| 错误情况 | 处理方式 |
|----------|----------|
| 视频文件损坏 | 提示用户检查文件完整性 |
| URL无法访问 | 提示用户检查链接有效性 |
| 视频时长过长 | 建议用户裁剪或分段分析 |
| ffmpeg未安装 | 提供安装指引，建议使用浏览器方式作为备选 |
| 关键帧提取失败 | 尝试调整参数或使用浏览器截图方式 |
| 场景变化检测无结果 | 自动降低阈值重试，或切换到 basic/hybrid 模式 |
| 帧数超限截断 | 脚本输出截断警告，优先保留场景变化帧 |
