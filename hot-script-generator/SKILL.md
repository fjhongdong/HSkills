# 热榜服饰种草脚本生成器（一键生成入口）

根据抖音热榜内容和角色形象，生成15-30秒的短视频脚本，让服饰自然成为故事的一部分。

**本技能为协调器，内部调用三个子技能完成完整流程。**

---

## 【三条核心铁律】（全流程不可突破）

### 铁律一：种草核心铁律
- **全程无服装特写镜头**，服装仅作为故事背景元素
- 服装永远服务于人物与剧情，不得作为画面主体
- ❌ 禁止：服装细节特写、标签展示、质地特写、旋转展示
- ✅ 正确：中远景人物情绪，服饰作为画面自然组成部分

### 铁律二：爆款适配铁律
- 严格适配**抖音9:16竖屏格式**
- **前3秒必须设置强钩子**，抓住用户注意力
- 单条视频时长**严格控制在15-30秒**
- 镜头数量控制在4-8个

### 铁律三：审美合规铁律
- 严格遵循**东方正向审美原则**
- 禁止：过度暴露、负面价值观引导、焦虑贩卖、刻板印象强化
- 提倡：自信、独立、温暖、治愈、积极向上的价值观

## 【子技能调用规范】

### 调用 hot-spot-analyzer

```
触发条件：解析热点信息后
调用方式：Skill 工具
传递参数：
  - hotspot_title: 热点标题
  - hotspot_content: 热点内容描述（可选）

预期输出（JSON）：
{
  "hotspot_info": { "title", "summary", "emotion_tone" },
  "emotion_analysis": { "surface_emotions", "deep_emotions", ... },
  "theme_analysis": { "topic_phenomenon", "underlying_issue", ... },
  "deduction": { "scene", "style", "storyline", "character_state" },
  "creation_suggestions": { "recommended_script_type", "recommended_hook_type" }
}
```

### 调用 character-fashion-analyzer

```
触发条件：热点分析完成后
调用方式：Skill 工具
传递参数：
  - fashion_image: 服饰图片（如有）
  - fashion_description: 服饰描述
  - emotion_tone: 热点分析的情绪基调
  - deduction_scene: 热点分析的推演场景

预期输出（JSON）：
{
  "character_info": { "gender", "age_group", "temperament" },
  "fashion_analysis": { "category", "color", "material", "style" },
  "audience_profile": { "target_group", "life_scenes", ... },
  "scene_adaptation": [ { "scene", "match" }, ... ],
  "season_adaptation": { "fashion_season", "match_result" },
  "creation_constraints": { "scene_constraints", "emotion_constraints", ... }
}
```

### 调用 script-creator

```
触发条件：角色形象分析完成后
调用方式：Skill 工具
传递参数：
  - hotspot_analysis: 完整的热点分析报告（JSON）
  - character_analysis: 完整的角色形象分析结果（JSON）
  - parameters: 可选参数（时长偏好、风格偏好等）

预期输出：
  - 完整脚本（Markdown）
  - 质量检查报告
  - 配套输出（BGM、标题、标签、发布建议）
```

## 【热点-角色匹配校验】

在调用 script-creator 之前，必须进行匹配校验：

| 校验项 | 说明 | 不匹配处理 |
|--------|------|------------|
| 热点推演场景 vs 服饰场景 | 场景是否兼容 | 以服饰场景为主调整 |
| 热点情绪基调 vs 服饰调性 | 情绪是否协调 | 以服饰调性为主调整 |
| 题材类型 vs 服饰风格 | 风格是否匹配 | 调整脚本类型 |

**校验通过条件**：
- 至少有一个适配场景匹配度 ≥ 中
- 情绪基调与服饰调性不冲突
- 题材类型与服饰风格兼容

**校验不通过处理**：
1. 向用户说明不匹配的具体原因
2. 提供调整建议（更换热点、更换服饰、或调整脚本方向）
3. 如用户坚持，则以服饰约束为主进行创作

## 【使用示例】

### 示例输入

```
热点：独居生活治愈瞬间
角色参考：穿着米色慵懒风卫衣的女生，气质温柔
```

### 执行流程

```
Step 1: 调用 hot-spot-analyzer
  → 情绪定位：治愈中带自足
  → 推演场景：阳光洒落的窗边角落
  → 推荐脚本类型：氛围感/OOTD

Step 2: 调用 character-fashion-analyzer
  → 服饰风格：慵懒家居风
  → 适配场景：居家（高）、周末休闲（中）、通勤（低）
  → 季节适配：秋季
  → 创作约束：场景=居家/周末休闲，情绪=治愈/温暖

Step 3: 匹配校验
  → 场景匹配：阳光窗边 ✓ 居家 ✓ 匹配
  → 情绪匹配：治愈 ✓ 温暖 ✓ 匹配
  → 风格匹配：氛围感 ✓ 慵懒风 ✓ 匹配

Step 4: 调用 script-creator
  → 生成脚本：18秒，4镜头，氛围感/OOTD
  → 质量检查：85分，良好
  → 配套输出：BGM建议、标题建议、话题标签

Step 5: 整合输出
  → 完整脚本报告
```

## 【子技能独立性说明】

三个子技能可独立使用：

| 子技能 | 独立使用场景 |
|--------|--------------|
| **hot-spot-analyzer** | 仅需热点分析，无需生成脚本 |
| **character-fashion-analyzer** | 仅需分析服饰/角色，无需关联热点 |
| **script-creator** | 已有分析结果，仅需生成脚本 |

**调用方式**：
- 一键生成：直接使用 `hot-script-generator`
- 分步执行：依次调用三个子技能，可插入人工审核环节
